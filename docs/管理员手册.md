## 🔐 系统管理

### 用户管理

进入 **后台管理** → **用户管理**：

- **新增用户**：创建店长/店员账号，分配门店
- **重置密码**：帮助忘记密码的用户重置
- **启用/禁用**：控制账号状态

### 门店管理

- 查看/编辑门店信息
- 设置门店排序
- 管理门店状态

## 📊 目标管理

### 设置月度目标

进入 **销售进度** → **目标管理**：

1. 选择年份和月份
2. 为每个门店设置：
   - 📱 手机销售目标（元）
   - ⌚ NCME 目标（元）
   - 📦 手机台量目标
   - ⭐ 重点机型目标（台）
   - 🔧 配件目标（元）
   - ♻️ 回收率目标
3. 点击「批量保存」

### 查看完成进度

销售进度页面以热力图展示各门店各指标完成率：
- 🟢 绿色：完成率 ≥ 80%
- 🟡 黄色：完成率 50%-80%
- 🔴 红色：完成率 < 50%

## 💾 数据管理

### 数据库备份

```bash
# 手动备份
python scripts/backup.py backup

# 查看备份
python scripts/backup.py list

# 恢复备份
python scripts/backup.py restore --file <文件>
```

系统每日凌晨 3:00 自动备份，保留最近 7 天。

### 数据导出

- **销售数据**：销售进度页面可导出 Excel
- **会员数据**：会员管理页面支持数据导出（即将上线）

## ⚙️ 系统配置

### 环境变量（.env 文件）

```bash
# 修改端口
SAMSUNG_PORT=9527

# 修改 JWT 密钥
SAMSUNG_SECRET_KEY=your-secret

# 修改 eBoss 目录
SAMSUNG_EBOSS_DIR=C:\eBoss\Local

# 禁用定时任务（调试用）
SAMSUNG_SCHEDULER_ENABLED=false

# 修改日志级别
SAMSUNG_LOG_LEVEL=DEBUG
```


## 🚨 故障排查

### 服务无法启动

1. 检查端口是否被占用：`netstat -ano | findstr 9527`
2. 检查数据库是否损坏：运行 `fix_and_start.py`
3. 查看日志：`logs/samsung-ops.log`

### 数据库修复

```bash
# 方式1：自动修复
python fix_and_start.py

# 方式2：手动修复
del data\samsung_ops.db-wal
del data\samsung_ops.db-shm
python -c "import sqlite3; db=sqlite3.connect('data/samsung_ops.db'); db.execute('PRAGMA journal_mode=DELETE'); db.commit(); db.close()"
```

### 性能优化

- 定期清理日志文件（`logs/` 目录）
- 定期清理旧备份（`samsung_ops_backup/` 目录）
- 数据库超过 100MB 时考虑归档历史数据
